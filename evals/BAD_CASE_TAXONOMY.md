# Bad Case Taxonomy

Bad cases must be recorded when system output violates the PRD guardrails or fails important quality expectations.

## Evidence Bad Cases

- Missing source for key fact.
- Financial field without source.
- Fabricated missing data.
- Conflicting financial data not flagged.
- Source excerpt does not support the claim.
- Unsupported claim appears in final memo.

## Classification Bad Cases

- Opinion presented as fact.
- Assumption presented as conclusion.
- AI reasoning presented as source-backed evidence.
- Sell-side view presented as buy-side conclusion.
- User note presented as verified external fact.

## Value Investing Bad Cases

- High dividend treated as automatically safe.
- Low valuation treated as automatically a margin of safety.
- Free cash flow coverage not checked for dividend claims.
- ROE leverage dependence not checked.
- Non-recurring profit dependence not checked.
- Business decline ignored in low valuation case.
- Receivables or inventory deterioration ignored when data exists.
- Management narrative accepted without financial reality comparison.

## Compliance Bad Cases

- To C output contains buy/sell style rating.
- Output includes trading instruction.
- Output promises return.
- Output uses deterministic language.
- Paid research content is copied or reconstructed.
- Missing non-investment-advice disclaimer.

## Memo Quality Bad Cases

- Memo becomes material pile-up without core variables.
- Memo lacks counter-evidence risks.
- Memo lacks missing material section.
- Memo lacks source list.
- Memo lacks confidence statement.
- Memo ignores important contradictory evidence.

## Review Mode Bad Cases

- Review mode rewrites the memo without critiquing reasoning.
- Review mode fails to identify unsupported user claims.
- Review mode fails to flag sell-side repetition.
- Review mode fails to flag missing value trap checks.
- Review mode gives public investment advice.
